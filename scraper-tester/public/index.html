Extraction

i
— disabled — — skip extraction. Use Raw HTML / Screenshot to get the page as-is.
CSS — pull structured data with CSS selectors. Uses lxml.cssselect.
XPath — same but with XPath expressions. More powerful for conditional / positional queries.
Click + Add Field for each piece you want. Every field row takes:
• field name — the key you will see in the response data object (e.g. title, price).
• selector — the CSS / XPath that points at the element (h1, .price_color, //h1).
• attr — what to pull out: text (text content), html (outer HTML), or an attribute name like href / src.
• all — off: first match as a string. On: every match as a list.
Example — real site: scrape app.cyberyozh.com
Fields: page_title=title/text, h1=h1/text →
data = {"page_title": "CyberYozh App", "h1": "..."}
Without extraction the same scrape only returns data: null plus raw HTML you would have to parse yourself.
all example: selector a on a page with 3 links
• off → data.link = "https://a"
• on → data.links = ["https://a","https://b","https://c"]

Custom Headers

i
Extra HTTP headers sent with the request.
Presets fill a common Chrome/Firefox/Safari/Mobile/RU/anti-bot fingerprint in one click. Picking a mobile preset also flips the Device selector to mobile so viewport + UA stay consistent.
Interaction with proxy geo: when proxy_geo.country_code is set, the scraper automatically picks a matching Accept-Language (e.g. es-ES,es;q=0.9 for Spain). Your custom headers are merged on top — any header you set here wins, including Accept-Language if you override it. Locale and timezone still follow the proxy geo.
Common additions: Referer, Cookie (prefer the Cookies section), Authorization, site-specific headers. Avoid setting User-Agent in both a preset and manually.

URLs

i
Parallelism — pages inside a single batch run in parallel using asyncio + the scraper's worker pool.
Default: WORKERS=2 browser processes, so up to 2 pages at a time per batch.
To increase: edit the scraper's .env file and bump WORKERS (each browser process uses ~300-500 MB RAM). Then restart the scraper with docker compose down && docker compose up -d.
Note: docker compose restart does not reload .env — use full down/up.
